
 Q 09. 예외처리(Exception Handling)에 대한 설명으로 바르지 않은것은?
	(1) 예외처리가 탐지되면 프로그램을 즉시 중단한 뒤 예외를 처리하고 다시 정상 실행한다.
	(2) 예외를 처리하는 부분을 예외처리기 라고 한다.
	(3) 예외상황이 탐지되면 프로그램 중단없이 적절한 행동을 취한 후 정상 실행한다.
	(4) 프로그램 실행중의 오버플로나 언더플로, 0으로 나누기 등으로 예외가 발생한다.
	
	// (1) 프로그램 중단하지 않음
	// (2) O
	// (3) 프로그램 중단없이, 적절한 행동 하고 정상실행 -- O
	// (4) OutOfBounds~~, Arithmetic~~ 이런 예외 발생함
	
Q 10. 다음중 자바의 예외처리에 대한 설명으로 옳은것은?
	(1) throw는 처리가 정상적으로 이루어졌음을 알리는 명령어이다
	(2) 예외가 발생할 가능성이 있는 문장은 try 블록에 넣는다
	(3) catch블록에는 예외발생 여부에 관계없이 실행할 문장을 넣는다
	(4) 발생한 예외에 대한 처리는 finally블록에서 할수 없다
	(5) 예외가 발생했으나, 예외처리가 이뤄지지 않은경우에 그 예외는 무시되고, 
		프로그램은 예외발생 이후 지점부터 계속된다.
		
	// (1) throw 던지는거임
	// (2) O 킹능성
	// (3) finally블록에 예외발생 여부 관계없이 실행할 문장 넣는거임
	// (4) 할수 있음 (무조건 실행하는거니까)
	// (5) 예외처리가 이뤄지지 않는경우에는 프로그램 오류이기 떄문에 프로그램이 종료된다.
